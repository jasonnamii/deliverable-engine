# Paper Engine (페이퍼엔진) — NYT 스타일 산출물 허브

산출물(deliverable·보고서·기획안·제안서·진단서·전략서·리포트·컨설팅보고서·프로파일·처방서) 품질 중앙 허브. 산문체 금지, NYT스타일(역피라미드·압정구조) 강제. 불릿·헤더·수치·인용·단문 필수.

---

| # | 규칙 | 위반시 |
|---|------|--------|
| 1 | **게이트키퍼** — md/html 생성 시 cascade 자동 발동 | 품질 붕괴 |
| 2 | **산문체 FORBIDDEN** — 긴 문장·연결어·설명형 문단 금지. 불릿·헤더·인용·수치·단문 필수 | NYT 스타일 위반 = FAIL |
| 3 | **역피라미드 강제** — 결론 1st. 뒤에서 30% 삭제 테스트 생존 필수 | 귀납구조 = FAIL |
| 4 | **3패스 삭제 의무** — 형용사·부사·연결어·중복 제거 후 산출 | 지방 잔존 |

| 7 | **디자인 위임** — design-skill 단일 권위. 구조·블록·QC만 | 역할 중복 |

## §A. MODE_ROUTER

입력 길이·용도로 분기.

| 길이 | 모드 | 구조 |
|---|---|---|
| ≤500자 | **MODE_S** | UP §6 3블록 (CONCLUSION·CASE·GROUND) 직행 |
| 500~2000자 | **MODE_M** | NYT 리드 + 압정(pin) |
| >2000자 | **MODE_L** | NYT 풀구조 (Headline·Lead·Nut·Body) |
| 용도 "상세·풀버전" | MODE_L 강제 | — |

---

## §B. NYT_STRUCTURE (MODE_M·L 핵심)

### B-1. 4층 구조

- **Headline** = 1줄, 결론 주장문 (예: "KISAS 플랫폼 3Q 목표 2배 상향")
- **Lead** = 1문단, 5W1H 또는 결론+핵심근거+핵심사례 (≤5문장)
- **Nut graf** = 1문단, "왜 이 문서가 중요한가" (MODE_L만)
- **Body** = 소제목 + 1~2문단 섹션, 중요도 내림차순

### B-2. 압정(pin) 비율

- 머리(Headline+Lead) : 몸통(Body) : 꼬리(결론) = **1 : 7 : 2**
- 도입·결론 각 1문단 상한, Body가 전체 70%+
- 별칭: 모래시계(구 v1.x) → 압정(v2.0)

### B-3. 자르기 테스트

- 뒤에서 30% 삭제 → 결론·핵심근거 생존? YES = PASS
- NO → 재배치 (중요도 내림차순 재정렬)

---

## §C. DELETION_FIRST_EDIT (산출 전 3패스 의무)

초안 작성 후 **반드시** 3패스 삭제 실행.

| 패스 | 삭제 대상 | 효과 |
|---|---|---|
| **pass1** | 형용사·부사 80% ("매우·정말·상당히·주로·기본적으로") | 분량 -20% |
| **pass2** | 연결어구 전부 ("그래서·따라서·즉·다시 말하면·정리하면·참고로") | 분량 -10% |
| **pass3** | 동의반복 (같은 뜻 다른 표현 나란히) | 분량 -10% |

목표: 원 draft 대비 **33~50% 감축**.

---

## §D. DENSITY_CHECK (산출 직전 QC 4항)

| # | 체크 | FAIL 조건 |
|---|------|-----------|
| 1 | **주제문 우선** | 문단 첫 문장이 주제문 아님 |
| 2 | **모호동사 빈도** | "있다·것이다·수 있다·볼 수 있다" > 문단당 1회 |
| 3 | **중복 금지** | 같은 정보 2회+ 등장 |
| 4 | **구체사실 밀도** | 문단당 인용·수치·고유명사 < 1개 (NYT 룰) |

1항이라도 FAIL → §C 재실행 후 재QC. 루프 하드캡 2회.

---

## §E. FORMAT (산문체 금지 강제)

### E-1. 강제 사용

- **불릿·번호·헤더** = 적극 활용 (스캔 가능성 우선)
- **인용 박스** = 중요 주장·숫자·고유명사 강조
- **표** = 비교·분류·매트릭스
- **수치** = "증가" 대신 "15% 증가", "많은" 대신 "3,200명"

### E-2. 금지

| 금지 | 교정 |
|---|---|
| 긴 문장 (25단어+) | 단문 2~3개로 분할 |
| 수동태 ("검토되었다") | 능동태 ("검토했다") |
| 모호동사 ("영향을 미친다") | 구체동사 ("15% 감소시킨다") |
| 연결어 남발 | 불릿·문단 분할로 대체 |
| 설명형 서술 ("~라고 할 수 있다") | 단언형 ("~다") |

## §F. Cascade 3티어 (기존 유지, 간소화)

용도 키워드("팀한테·보고·제출·보내") → 외부, 나머지 → 내부.

| | ≤20행 | 21~50행 | 51+행 |
|---|:-:|:-:|:-:|
| 외부 | 경량 | 풀 | 풀 |
| 내부 | 패스스루 | 경량 | 풀 |

- **풀** = §A~§E 전량 + 4축 QC
- **경량** = §A·§C·§D·§E (구조 스킵)
- **패스스루** = §E만 (FORMAT만)

**§F-1 의도 게이트:** 문서 의도 1% 감지 → 1문장 제안, P2 동사 명시 → 즉시 §A.

### 시각소스 감지 훅 (HTML 전용)

- Body 섹션 설계 중 **V1~V10 태깅**: 수치비교·시간축·프로세스·순환·2축분류·핵심수치·비율·퍼널·관계망·매트릭스
- 산출 직전 design-skill에 블록별 시각소스 목록 전달 → C9 강제 발동
- 상세: `references/cascade-protocol.md §12`

## §G. 파이프라인

```
① PREFLIGHT(수신자·목적·핵심메시지) → ② MODE_ROUTER → ③ NYT_STRUCTURE 작성
→ ④ DELETION_FIRST_EDIT(3패스) → ⑤ DENSITY_CHECK(4항) → ⑥ FORMAT 검증
→ ⑦ [HTML] 시각소스 design-skill cascade
→ ⑧ [HTML] 모바일 QC 게이트 — `bash mnt/.claude/skills/design-skill/scripts/qc-mobile.sh output.html`
  · R2·R4b·R8·R9·R10·R11·R1a 정적 grep 검사
  · FAIL 시 design-skill responsive.md 참조 후 재수정. 루프 하드캡 2회.
→ ⑨ 산출
```

QC 없이 전달 = FAIL. 모바일 375px 실측 미검증 = FAIL.

## Gotchas

| 함정 | 대응 |
|---|---|
| 산문체 관성 | §E 강제. 불릿·단문·수치로 재작성 |
| 결론이 끝에 (귀납) | §B 역피라미드. Headline 1st |
| 형용사·부사 남발 | §C pass1 80% 삭제 |
| 문단당 수치·인용 0개 | §D-4 FAIL. 구체사실 주입 |
| 긴 문장 25단어+ | §E-2 단문 분할 |
| 수동태 "검토되었다" | §E-2 능동태 |
| 모호동사 "영향 미친다" | §E-2 구체동사 "15% 감소" |
| HTML 시각소스 미전달 | §F V1~V10 태깅 + design-skill C9 cascade |
| HTML 모바일 QC 누락 | §G-⑧ qc-mobile.sh 필수. 데스크톱만 보고 제출 = 모바일 깨짐 수정 루프 3~5회 발생 |
| cascade 미적용 | md/html 생성 시 자동 발동 |
| 내부 라벨 노출 | STEALTH (§·MODE·ⓐ 본문 ✗) |
| 모래시계 명칭 사용 | v2.0부터 "압정(pin)". 별칭 허용, 신규 문서는 압정 |
| UP §6 충돌 | UP §6은 대화 출력, paper-engine은 문서. MODE_S에서만 §6 3블록 공유 |
