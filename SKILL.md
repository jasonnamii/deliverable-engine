# Deliverable Engine

모든 도메인스킬의 .md 산출물을 능동적으로 받아 구조·블록·QC를 적용하는 **중앙 허브**. 도메인스킬은 이 엔진을 이름으로 부르지 않는다 — 사용자의 출력 포맷 키워드("보고서로", "진단서로", "리포트로" 등)로 자동 발동되거나, .md 산출물 생성 시 상시 cascade 적용된다. 제출청소(외부 제출 전 13축 정제)는 trigger-dictionary 관할.

---

## §0. Cascade — 중앙 허브

**핵심 규칙:** 어떤 스킬이든 .md 산출물 파일을 생성하면, 그 출력 시점에서 §1~§4가 자동 적용된다. 별도 Skill tool 호출 불필요 — 상시 적용. 도메인스킬이 이 엔진을 명시적으로 호출하거나 언급할 필요 없다.

### 3티어 게이트 — 용도 × 길이 매트릭스

**용도 판정:** "팀한테/보고/공유/제출/보내" → 외부. 나머지 → 내부. 판정 불가 → 외부(안전측).

| | 20행 이하 | 21~50행 | 51행+ |
|---|:---:|:---:|:---:|
| **외부 전달용** | 경량 + QC(숫자+VOCAB만) | 풀 적용 | 풀 적용 |
| **내부/개인용** | 패스스루(1블록1목적만) | 경량 | 풀 적용 (ⓕ디자인 선택적) |

- **풀:** §1 파이프라인 전체 (ⓐ~ⓔ) + QC 4축 전수
- **경량:** 1블록1목적·여백구분 + QC(①숫자+③VOCAB만)
- **패스스루:** 1블록1목적 확인만. QC·디자인 스킵
- 대화 중 **문서 의도 감지** (§0-A) → 선제 전환 → 수락 시 해당 티어 적용
- 순수 대화 (문서 의도 0%) → 미적용

### §0-A. 문서 의도 감지 — 1% 게이트

**원칙:** 대화는 대화로 유지한다. 단, 문서 의도가 1%라도 감지되면 선제적으로 문서화 전환을 제안한다. 제안 = 1문장. 수락 전까지 대화 모드 유지.

**감지 신호 (하나라도 해당 → 발동):**

| 유형 | 신호 예시 |
|------|----------|
| 명시적 기록 의도 | "정리해볼까", "기록으로 남기자", "나중에 볼 수 있게", "문서로 남기면" |
| 암묵적 축적 | 분석·논의가 **3턴 이상 누적** + 구조화 가능한 결론 존재 |
| 외부 전달 암시 | "이거 팀한테", "공유하려고", "보내려면", "보고해야" |
| 포맷 힌트 | "표로", "깔끔하게", "한눈에", "요약하면" (단, 대화 내 간단 요약 요청은 제외) |
| 반복 참조 가능성 | "다음에도 쓸", "템플릿으로", "매번 이렇게" |

**발동 프로토콜:**
1. 신호 감지 → 현재 응답을 **대화 톤으로 완성** (문서화하지 않음)
2. 응답 말미에 1문장 제안: "문서로 정리해드릴까요?" (변형 허용, 핵심 = 짧고 부담없는 제안)
3. 수락 → §1 풀 파이프라인 진입. 거절/무반응 → 대화 모드 유지, 같은 대화에서 동일 신호 재제안 ✗

**P2 동사 우선 규칙:** P2 동사("작성해줘", "만들어줘", "정리해줘", "써줘" 등)가 명시된 요청 → §0-A 스킵, 즉시 §1 진입. §0-A는 P2 동사 **없이** 포맷 힌트만 있을 때만 발동.

**미발동 = FAIL 조건:** 위 신호가 존재하는데(P2 동사 미포함) 제안 없이 대화만 이어간 경우.
**과잉발동 방지:** 순수 대화(의견 교환, 질문-답, 잡담)에서 신호 없이 제안 = 과잉. 1세션 내 거절 후 동일 주제 재제안 ✗.

**적용 순서 (5단계 — ⓑⓒ 병합):** 타 스킬 분석 완료 → ⓐ spine 추출(결론 1문장) → ⓑ 모래시계+블록설계 동시 작성(spine 배치·밀도교대·전환장치를 1회 작성에서 통합) → ⓒ 제목 변환 → ⓓ QC(티어별 범위) → ⓔ 디자인(design-skill cascade, 풀 적용시만 필수)

**ⓑ 통합 작성 원칙:** "모래시계 구조로 배치하면서, 동시에 블록 밀도를 교대 배치하라"는 단일 지시로 실행. 개념적으로는 §2(매크로)와 §3(메조)가 분리되지만, 실행은 1회 작성에서 동시 적용. 2패스 재작성 ✗.

**spine 추출 실패 폴백:** spine 추출이 불가능한 경우(순수 데이터, 리스트 온리, 다중 동등 결론) → **참조형 문서로 자동 분류**. 모래시계 면제, 라벨형 제목 허용, 블록설계+QC만 티어에 따라 적용. → references/cascade-protocol.md "spine 추출 실패 폴백" 참조

**타 스킬 책임 분리:** cascade는 **출력 형태만 변환**. 분석 로직·프레임·도메인 지식은 타 스킬 관할.

**디자인 위임:** 산출물의 시각 디자인은 **design-skill이 단일 권위**. 이 엔진은 구조·블록·QC만 담당하고, .md 파일 생성 시 design-skill을 cascade 호출하여 시각문법을 적용한다. 콜아웃 팔레트·HTML 스타일·톤 매핑 등을 이 엔진이 독자적으로 결정하지 않는다.

**VOCAB (내부용어 치환):** 이 엔진이 유일한 소스. 모든 도메인스킬 산출물에서 내부용어를 일반어로 치환하는 규칙은 §1-③ QC에서 일괄 적용.

**상세 절차·변환 예시·스킬별 주의사항:** → references/cascade-protocol.md 참조

## §1. 3단계 파이프라인

```
①프레이밍 → ②작성 → ③QC (스킵 ✗)
```

**금지어 사전 적용:** 작성 단계에서 금지어 목록을 참조, 설계 시점에서 선제 회피.

### ①프레이밍
수신자 + 목적 + 핵심메시지 3개를 확정한 뒤 작성 시작.
- 추론 가능 → 자율확정 + 즉시착수
- 불가 → 최선가정 + 명시 후 진행
- 형이 3개를 이미 줬으면 게이트 자동통과
- **개인 문서(수신자=형):** 수신자 자동확정. 목적 + 핵심메시지만 확정하면 즉시착수
- **cascade 시:** 타 스킬의 분석 결론이 핵심메시지. 수신자·목적은 원래 요청에서 추론. **수신자 미추론이면 수신자=형 자동확정, 질문 ✗**

### ②작성
1. **매크로+메조 통합** — 모래시계(§2) + 블록설계(§3)를 1회 작성에서 동시 적용. 2패스 ✗
2. **마이크로** — 문체 (UP Core TONE 참조)

### ③QC — 티어별 범위

| 검사 | 풀 적용 | 경량 적용 | 패스스루 |
|------|:---:|:---:|:---:|
| ①숫자정합 | ✓ | ✓ | — |
| ②논리완결 | ✓ | — | — |
| ③문체·어휘(VOCAB) | ✓ | ✓ | — |
| ④블록설계정합 | ✓ | — | — |

- **풀:** 4축 전수. 외부 전달 21행↑, 내부 51행↑
- **경량:** ①+③만. 외부 20행↓, 내부 21~50행
- **패스스루:** QC 스킵. 내부 20행↓ (1블록1목적 확인만)

**VOCAB** — 산출물 내 금지어 치환:

| 원문 | 처리 |
|------|------|
| "spine" | → "핵심 메시지 정합성" |
| "MECE" | 삭제 |
| "triage" | → "우선순위 선별" |

**금지어 절대 규칙:** 산출물 파일 전체에서 금지어 원문 0개 필수. QC 확인 보고는 **"지정 어휘 치환 완료 ✓"** 1문장.

QC 없이 전달 = FAIL.

## §2. 매크로 구조 — 모래시계

```
역피라미드(핵심 먼저) → spine(가장 밀도 높은 구간) → 피라미드(세부 확장)
```

- spine에 종속되지 않는 내용 → 삭제
- 밀도 ∝ spine 근접도
- 1섹션 = 1메시지 (2메시지는 구조상 분리불가시만)
- 기획안: spine → 본문(핵심만) → 부록(접힘. 삭제 ✗, 본문비대 ✗)

## §3. 메조 구조 — 블록설계

정보는 선형나열이 아니라 의미단위(chunk)로 묶여야 인지된다.
✗ 워드문서식(제목라벨→균질본문→제목라벨 반복)

### 7규칙

| # | 규칙 | 판정 |
|---|------|------|
| 1 | **1블록1목적** | 블록당 커뮤니케이션 목적 1개. 2↑ 혼재→FAIL |
| 2 | **블록최소의존** | 단독으로 핵심의미 파악 가능. 논증체인 의존은 @ref로 허용 |
| 3 | **블록내맥락흐름** | 도입→본체→귀결 필수. 2↑누락→FAIL |
| 4 | **블록전환** | 인접블록 간 서사전환장치 의무. ENUM: 질문→답, 원인→결과, 긴장→해소, 전제→확장, 대비→수렴. 없으면→FAIL |
| 5 | **여백구분** | 블록경계 = 공간(빈행·hr·콜아웃경계). 시각분리 없이 직접연결→FAIL |
| 6 | **밀도대비** | 인접블록 밀도차이 필수. ENUM + 경계 판정 규칙(아래). **동일ENUM 3연속→FAIL** |

**밀도 ENUM:** 고(데이터·수치·비교표), 중(논증·분석), 저(서사·맥락·전환). 경계 판정 규칙(수치 3개↑→고 승격 등) → references/cascade-protocol.md "밀도 경계 판정" 참조
| 7 | **블록크기가이드** | ~35행 기준(가이드). 초과시 분할검토 1회(유지 허용) |

**50행↓ 문서 면제:** 블록전환·밀도대비·블록크기가이드 면제. 1블록1목적·여백구분만 적용.

### 축별나열 안티패턴 (cascade 핵심)

타 스킬의 축별 분석(biz-skill 18축, risk-radar 매트릭스 등)을 그대로 출력하면 **동일 밀도 N연속 나열** = 규칙4+6 동시 위반.

**변환 4원칙:** ①축을 인과·시간·논증 흐름의 서사로 엮어라 ②밀도를 교대하라(서사→표→논증) ③참조용 데이터는 부록으로 ④"~축에서도" 라벨 전환 금지→서사전환장치 강제

**상세 변환 규칙·Before/After 예시:** → references/cascade-protocol.md 참조

## Gotchas

| 함정 | 대응 |
|------|------|
| **타 스킬 뒤에서 cascade 미적용** | .md 파일 생성 시점에서 §0 자동 발동. 상시 적용 |
| **축별 나열을 그대로 출력** | §3 축별나열 안티패턴 → 서사 엮기 + 밀도 교대 |
| **결론이 문서 끝에** | §2 모래시계 — spine을 첫 3줄에. cascade ⓑ에서 재배치 |
| 프레이밍 스킵 | 수신자·목적 불명확한 채 작성 → 전량 재작업 |
| 균질 텍스트 벽 | 블록전환 없이 같은 밀도로 나열 → 가독성 급락 |
| 주장형/라벨형 혼동 | 기획안에 "1. 배경" 라벨 제목 → spine 전달력 저하 |
