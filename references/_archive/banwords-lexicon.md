# BAN_LEXICON — AI 흔적 사전 (v3.0 신규, INV 16)

산출물에서 grep 적발 시 차단·평문변환·재출력. 형 코퍼스 23개 PDF·1MB 실측으로 0회 검증된 어휘 위주.

본 사전은 SKILL.md INV 16의 단일 권위. `scripts/humanize_check.py`는 이 사전의 *보조*.

---

## 1. 본질 5줄

| | |
|---|---|
| **RULE** | AI가 흘리는 5대 흔적 단어 산출물 노출 = FAIL. 형 코퍼스 1MB 실측으로 0회 검증된 어휘 우선 차단 |
| **판정** | "이 단어, 형 코퍼스 1MB 안에서 한 번이라도 등장하나?" — NO → 차단 |
| **ALLOW** | 업계 전문용어 · 고유명사 · 법조문 · 직접 인용 |
| **CONVERT** | 적발 → 평문 풀어쓰기. 동의어가 아닌 *형 시그니처 어휘*로 재구성 |
| **SELF_CHECK** | 산출 직전 자체 grep. 1개라도 적발 = 차단·재작성 |

## 9. 자체검사 절차

```bash
# 1차: 사전 grep
python scripts/humanize_check.py {산출파일경로}

# 2차: LLM 자체 판정
"이 문장, 형 코퍼스 1MB 안에 한 번이라도 등장할 만한가?"
NO → 차단·재작성
```

**부분치환 금지.** 적발 단어만 바꾸지 말 것. 사고 자체가 AI식이어서 부분치환만으론 자연어가 안 됨. **전수 재작성**으로 사고 자체를 바꾼다.

---

## 10. 우선순위 — BAN_LEXICON vs 다른 규칙

BAN_LEXICON은 **최상위 게이트**. 다른 규칙(예: CEW 한 줄, 압정 비율)이 BAN 단어를 요구하면 그 규칙을 자연어로 풀어쓴다. BAN 단어가 통과하는 일은 없다.

## 13. 카테고리 9 — UP BAN DICTIONARY (v3.3 신규, INV 24)

형 UP v122.1 BAN DICTIONARY 4섹션 verbatim 박제. SCOPE: 산출물 + 6줄+ 본문 + CDB.

상세 verbatim: `→ references/up-ban-dictionary.md`

### 13-1. 작업프레임 BAN (요약)
- 숫자카운터: `[0-9]+ + (축|건|대|종|개|차원|영역|항목|측면|관점|종류|범주|갈래|방면|국면|요소|구분)`
- 분석프레임 BAN: 축별·다축·수렴축·차원이·차원에서·영역에서·관점에서·측면에서·종류로·범주로·항목으로·구분으로
- 메타코드 BAN: Phase N·P1·P2·MUST·SHOULD·MAY·SCOPE_OUT·SCOPE_IN·LIGHT·DEEP·TURBO·인물 동학·조직 동학·경력 DNA·조직 DNA·§AI·§H·자체검사·자체점검
- 예외: "DNA" 단독 면제

### 13-2. AI어휘 BAN (요약)
- 메타어휘: 박제·박다·박혀·박힌·박혀있·새기다 → 저장·기록·등록·고정·넣다·두다·적다
- 미사여구: 혁신적·차세대·포용적·전사적·윈윈·유기적·역동적·다각도로·다층적으로·전례 없는·지속 가능한·차별화된·함께 만들어가는
- 컨설팅투: 솔루션 기반·전략적 파트너·고객 중심·체계적이고·필수적인·선도적인·전략적 우위·고객 가치 극대화
- 관계UX: 원활한·긴밀히·적극적인·종합적인·건설적·지속적인·효과적인·사용자 친화적·직관적이고·최적화된·원활한 경험·효율적인·사용자 경험을 향상
- 약 BAN (hit≥2 = 재작성): 여러분의·진정한·또한·그러므로·한편·유연하게

### 13-3. 자가합리화 차단 FAIL 마커
"프레임이라 유지"·"숫자 없으니 OK"·"프레임이라 필요"·"이번엔 예외"·"일상어"·"의미상 OK"·"메타라 OK"

### 13-4. self-check (요약)
- 허용: 업계 표준(BEP·KPI·MECE·MVP)·고유명사·법조문번호
- SCOPE: 산출물 + 6줄+ 본문 + CDB
- GOOD: 마케팅·콘텐츠·브랜드·캠페인·플랫폼·새로운·다양한·때문에·다이렉트·퍼포먼스·타겟팅
- 스타일: 단문 8~25자·능동형·구체명사 우선
- CHECK: 송출 직전 grep 미수행 = FAIL · hit≥1 → 변환·재출력 · 미탐 사후 = 형 ping

## 14. 카테고리 10 — KIWI 글쓰기 5법칙 BAN (v3.4 신규, INV 25)

KIWI Company 글쓰기 PDF 5법칙 통합. 한국 명문 산문 미학 강제. 상세 정본: `→ references/kiwi-writing-rules.md`

### 14-1. '것' 남용 BAN (KIWI 2-1)
| BAN 패턴 | 변환 |
|---|---|
| ~하는 것이다 | ~한다·~이다 |
| ~하는 것은 | ~함은·~하기는 |
| ~한다는 것은 | ~한다는 점은·~한다는 사실은 |
| ~책이라는 것 | ~책 |

**검출:** 한 문장 '것' ≥ 2 = WARN, ≥ 3 = FAIL

### 14-2. '도·등' 자주 쓰지 말라 BAN (KIWI 2-2)
| BAN 패턴 | 변환 |
|---|---|
| A도 B도 C도 (3+ 연속 '도') | A가·B도·C 역시 (어미 변주) |
| A·B·C·D 등 | A·B·C·D처럼 / 같은 이들 |

**검출:** 한 문단 '도' ≥ 3 = WARN

### 14-3. 자신없는 표현 BAN (KIWI 3-3 — 단정형 강제)
| BAN | 변환 |
|---|---|
| ~인 것 같다 | ~이다 / ~인 듯하다 |
| ~인 것 같습니다 | ~입니다 |
| ~라고 한다 | ~이다 |
| ~인 것이다 | ~이다 |
| ~이 아닌가 싶다 | ~이다 |
| ~할 것인 것 같다 | ~할 것이다 / ~할 듯하다 |
| 언젠가 (시간 모호) | 1976년·작년·구체 |
| 어디선가 | 구체 장소 |

**검출:** grep '인 것 같|라고 한다|인 것이다|아닌가 싶' = HIGH FAIL

### 14-4. 한자투 표현 BAN (KIWI 4-1)
| BAN | 변환 |
|---|---|
| ~로 인하여 | ~로·~때문에 |
| ~에 있어 | ~에서·~에·~로 |
| ~함에 있어 | ~할 때·~하면서 |
| ~로 인해 | ~로·~때문에 |
| 탑승하고 있던 | 탄 |
| ~하기에 ~역부족 | ~기엔 부족 |
| 현 상황 속에서 | 지금·요즘 |

**검출:** grep '로 인하여|에 있어|함에 있어' = HIGH

### 14-5. 강조 부사 BAN (KIWI 1-5)
**기존 §3 형식부사 BAN 강화. 형 시그니처 부사(정말·확실히·진짜·딱·솔직히)는 ALLOW.**

| BAN 강조 부사 | 변환 |
|---|---|
| 매우 | (삭제) |
| 너무나 | (삭제) |
| 대단히 | (삭제) |
| 굉장히 | (삭제) |
| 엄청나게 | (삭제) |
| 무척 | (삭제) |

**ALLOW (형 시그니처 면제):** 정말·확실히·진짜·딱·솔직히·사실·근데·그냥·왜·어떻게

### 14-6. 빼도 좋을 조사 (KIWI 4-4)
**선택적 BAN — 산문 산뜻하게 다듬을 때 적용. 의미 모호 시 유지.**

| 빼면 좋은 패턴 | 변환 |
|---|---|
| 이름 아래에 | 이름 아래 |
| 사람의 마음 | 사람 마음 |
| 밤 10시 이후에 | 밤 10시 이후 |
| 진저리를 치는 | 진저리 치는 |
| 말이 없는 | 말 없는 |

### 14-7. 과잉 감정 표현 BAN (KIWI 4-3)
| BAN | 변환 |
|---|---|
| 제일 먼저 | 먼저 |
| 것은 물론 | (삭제) |
| ~할 정도로 (감정 강조) | (삭제 또는 사실로) |
| 심지어 (감정 강조) | (삭제 또는 사실로) |
| 마치 ~처럼 (과잉 비유) | (사실로) |
