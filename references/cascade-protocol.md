# Cascade Protocol — 타 스킬 후속 발동 + 세부 구조 규칙

산출물 엔진이 타 스킬 뒤에서 자동 적용될 때의 절차, 매크로·메조 구조의 세부 규칙, VOCAB 치환 규칙.

---

## 1. Cascade 발동 판정

| 조건 | 발동 | 이유 |
|------|------|------|
| 타 스킬이 .md 산출물 파일을 생성 | **자동 적용** | 산출물=구조화된 서술 |
| 타 스킬이 대화 내 인라인 응답만 | 미적용 | 파일 미생성 |
| 타 스킬이 .xlsx/.pptx 등 비md | 미적용 | 해당 포맷 스킬 관할 |
| 타 스킬 산출물 | **3티어 게이트** | 허브 §0 용도×길이 매트릭스 참조 |

**"자동 적용"의 의미:** 타 스킬의 분석·진단이 완료된 후, 그 결과를 .md 파일로 출력하는 시점에서 매크로(§2) + 메조(§3) + 제목규칙(§4)을 적용한다. 타 스킬의 분석 로직을 변경하는 것이 아니라, **출력 형태를 변환**한다. 적용 범위는 허브 §0 3티어 게이트(용도×길이)에 따라 풀/경량/패스스루로 결정.

## 2. 적용 순서 (5단계 — ⓑⓒ 병합)

```
타 스킬 분석 완료 → ⓐ spine 추출 → ⓑ 모래시계+블록설계 통합 작성 → ⓒ 제목 변환 → ⓓ QC → ⓔ 디자인
```

| 단계 | 할 일 | 구체적으로 |
|------|-------|-----------|
| ⓐ spine 추출 | 분석 결과에서 핵심 판정 1문장 도출. **실패 시 → 참조형 폴백** | "이 문서의 결론은 무엇인가" — 그것이 spine |
| ⓑ 모래시계+블록설계 통합 | spine 배치 + 밀도교대 + 전환장치를 **1회 작성에서 동시 적용** | 2패스 재작성 ✗. 단일 지시: "모래시계 구조로 배치하면서 블록 밀도를 교대 배치하라" |
| ⓒ 제목 변환 | 라벨형→주장형 (설득형 문서시) | "패턴 대조"→"6개 축 전부 실패 패턴과 매칭된다" |
| ⓓ QC | **티어별 범위** — 풀: 4축 전수 / 경량: ①숫자+③VOCAB만 / 패스스루: 스킵 | 허브 §1-③ 참조 |
| ⓔ 디자인 | design-skill cascade | 풀 적용시 필수. 내부/개인 문서는 형 요청시만 |

## 3. spine 추출 실패 폴백 — DOC_TYPE 분기로 통합 (v2.3)

spine 추출 시도 시 v2.3부터는 **DOC_TYPE 분기**(SKILL.md §A-2, references/doc-types.md)로 흡수.

| 실패 유형 | DOC_TYPE 매핑 | 처리 |
|----------|---------------|------|
| 순수 데이터 (표·수치만) | DELIVER (참조형) | 압정 1:7:2·매크로 면제·메조+QC만 |
| 리스트 온리 (체크리스트) | DELIVER | 라벨형 제목 허용·1블록1목적+여백구분 |
| 다중 동등 결론 (3개+ 병렬) | **EVALUATE** (압정 1:5:4) | 등급표 + 근거 컬럼 의무 (INV 10) |
| 다소스 통합 (3편+) | **CONVERGE** (압정 1:4:5) | 교차비교 매트릭스 의무 (F-2) |

**v2.2 이전 "참조형 폴백":** v2.3부터 DELIVER(참조형)·EVALUATE·CONVERGE로 세분화. 매크로 면제는 DELIVER만, EVALUATE/CONVERGE는 본문 비중 ↑로 사고형 산출물 보장.

## 4. §0-A 문서 의도 감지 — 1% 게이트 (세부)

**원칙:** 대화는 대화로 유지한다. 단, 문서 의도가 1%라도 감지되면 선제적으로 문서화 전환을 제안한다. 제안 = 1문장. 수락 전까지 대화 모드 유지.

**감지 신호 (하나라도 해당 → 발동):**

| 유형 | 신호 예시 |
|------|----------|
| 명시적 기록 의도 | "기록으로 남기자", "나중에 볼 수 있게", "문서로 남기면" |
| 암묵적 축적 | 논의가 **3턴 이상 누적** + 구조화 가능한 결론 존재 |
| 외부 전달 암시 | "이거 팀한테", "공유하려고", "보내려면", "보고해야" |
| 포맷 힌트 | "표로", "깔끔하게", "한눈에", "요약하면" (단, 대화 내 간단 요약 요청은 제외) |
| 반복 참조 가능성 | "다음에도 쓸", "템플릿으로", "매번 이렇게" |

**발동 프로토콜:**
1. 신호 감지 → 현재 응답을 **대화 톤으로 완성** (문서화하지 않음)
2. 응답 말미에 1문장 제안: "문서로 남겨드릴까요?" (변형 허용, 핵심 = 짧고 부담없는 제안)
3. 수락 → 허브 §1 풀 파이프라인 진입. 거절/무반응 → 대화 모드 유지, 같은 대화에서 동일 신호 재제안 ✗

**P2 동사 우선 규칙:** P2 동사("만들어줘", "써줘", "드래프트해줘" 등)가 명시된 요청 → §0-A 스킵, 즉시 §1 진입. §0-A는 P2 동사 **없이** 포맷 힌트만 있을 때만 발동.

**미발동 = FAIL 조건:** 위 신호가 존재하는데(P2 동사 미포함) 제안 없이 대화만 이어간 경우.
**과잉발동 방지:** 순수 대화(의견 교환, 질문-답, 잡담)에서 신호 없이 제안 = 과잉. 1세션 내 거절 후 동일 주제 재제안 ✗.

## 5. 매크로 구조 — 모래시계

```
역피라미드(핵심 먼저) → spine(가장 밀도 높은 구간) → 피라미드(세부 확장)
```

- spine에 종속되지 않는 내용 → 삭제
- 밀도 ∝ spine 근접도
- 1섹션 = 1메시지 (2메시지는 구조상 분리불가시만)
- 기획안: spine → 본문(핵심만) → 부록(접힘. 삭제 ✗, 본문비대 ✗)

## 6. 메조 구조 — 블록설계 7규칙

정보는 선형나열이 아니라 의미단위(chunk)로 묶여야 인지된다. ✗ 워드문서식(제목라벨→균질본문→제목라벨 반복).

| # | 규칙 | 판정 |
|---|------|------|
| 1 | **1블록1목적** | 블록당 커뮤니케이션 목적 1개. 2↑ 혼재→FAIL |
| 2 | **블록최소의존** | 단독으로 핵심의미 파악 가능. 논증체인 의존은 @ref로 허용 |
| 3 | **블록내맥락흐름** | 도입→본체→귀결 필수. 2↑누락→FAIL |
| 4 | **블록전환** | 인접블록 간 서사전환장치 의무. ENUM: 질문→답, 원인→결과, 긴장→해소, 전제→확장, 대비→수렴. 없으면→FAIL |
| 5 | **여백구분** | 블록경계 = 공간(빈행·hr·콜아웃경계). 시각분리 없이 직접연결→FAIL |
| 6 | **밀도대비** | 인접블록 밀도차이 필수. ENUM + 경계 판정 규칙(아래). **동일ENUM 3연속→FAIL** |
| 7 | **블록크기가이드** | ~35행 기준(가이드). 초과시 분할검토 1회(유지 허용) |

**밀도 ENUM:** 고(데이터·수치·비교표), 중(논증·해석), 저(서사·맥락·전환).

**50행↓ 문서 면제:** 블록전환·밀도대비·블록크기가이드 면제. 1블록1목적·여백구분만 적용.

## 7. 밀도 경계 판정

| 밀도 | 기본 구성 | 경계 승격 규칙 |
|------|----------|---------------|
| 고 | 데이터·수치·비교표 | — |
| 중 | 해석 | 수치 3개↑ 포함 → 고로 승격. 직접인용 2개↑ → 중 유지(저로 하락 ✗) |
| 저 | 서사·맥락·전환 | 인용 1개 포함 → 저 유지. 해석적 서사(인과 추론 포함) → 중으로 승격 |

## 8. 축별나열 → 블록전환 변환

타 스킬(biz-skill, risk-radar, research-frame 등)은 본질적으로 "축별" 결과를 낸다. 그대로 출력하면 **동일 밀도 N연속 나열** — 규칙6(밀도대비) 위반.

### 안티패턴: "사전식 나열"

```
## 패턴 대조                     ← 라벨형 제목
자본 축에서는 A다. (중밀도 해석)
플랫폼 축에서는 B다. (중밀도 해석)
규제 축에서는 C다. (중밀도 해석)
제휴 축에서는 D다. (중밀도 해석)
시장 축에서는 E다. (중밀도 해석)
→ 5연속 동일 밀도 = 규칙6 위반
→ 축 라벨 전환만 = 규칙4 위반
```

### 변환 4원칙

| # | 원칙 | 방법 |
|---|------|------|
| 1 | **축을 서사로 엮어라** | 인과·시간·논증 흐름으로 재배치. "자본이 없으면→플랫폼 불가→시장 신뢰 붕괴"처럼 축간 영향 맵이 서사의 뼈대 |
| 2 | **밀도를 교대하라** | 서사(저) → 데이터/표(고) → 해석(중) 교대. "서사도입→표→해석→서사전환→표→해석" |
| 3 | **부록으로 분리하라** | 전체 축 스크리닝 결과표, 매칭 근거표 등 참조용 데이터는 부록으로. 본문에는 핵심 축만 서사로 |
| 4 | **블록 전환 장치를 삽입하라** | 축과 축 사이에 "원인→결과", "전제→확장", "긴장→해소" 중 하나 강제. "~축에서도" "~측면에서 보면" 같은 라벨 전환 금지 |

### Before/After

**Before (축별 나열):**
```markdown
## 패턴 대조 서사
자본 구조가 부재하다. Zoom은... WeWork는... (6문단)
플랫폼 측면에서... Airbnb는... Quibi는... (6문단)
규제 축에서도... Google은... SEC는... (6문단)
```

**After (서사 엮기 + 밀도 교대):**
```markdown
## 6개 축이 하나의 실패 패턴을 가리킨다

이 문서에는 단위경제학이 없다. [결론 먼저 — 고밀도 1문장]

Zoom이 $1.8M 시드에서 $100B IPO까지 도달한 변수는 CAC payback 집착이었다.
이 문서는 그 개념 자체가 부재하다. [서사 — 저밀도]

| 축 | 판정 | 매칭 패턴 |
|...|...|...|  [표 — 고밀도, 핵심 3축만]

자본이 없으면 플랫폼을 만들 수 없고, 플랫폼이 없으면 시장 신뢰가 붕괴한다.
세 축이 상호 강화하는 하향 나선이다. [인과 엮기 — 중밀도]
```

## 9. VOCAB — 내부용어 치환 (유일 소스)

모든 도메인스킬 산출물에서 내부용어를 일반어로 치환하는 규칙. 허브 §1-③ QC에서 일괄 적용.

| 원문 | 처리 |
|------|------|
| "spine" | → "핵심 메시지 정합성" |
| "MECE" | 삭제 |
| "triage" | → "우선순위 선별" |

**금지어 절대 규칙:** 산출물 파일 전체에서 금지어 원문 0개 필수. QC 확인 보고는 **"지정 어휘 치환 완료 ✓"** 1문장.

## 10. 제목 규칙

| 산출물 유형 | 제목 스타일 |
|-------------|------------|
| 설득형(기획안·제안서·해석보고) | 주장형 ("X가 Y인 이유") — 라벨 ✗. "1. 배경" "2. 현황" 번호+라벨도 ✗. 모든 H2·H3가 주장·행동·질문 형태 |
| 참조형(매뉴얼·가이드·사전) | 라벨형 정답 |
| 혼합형 | 핵심메시지직결=주장형, 보조·부록=라벨 허용 |

## 12. 시각소스 감지 훅 (HTML·웹MD cascade 전용)

산출물 포맷이 HTML/웹MD일 때 ⓑ 블록 설계 과정에서 "시각소스" 블록을 식별해 태깅하고, ⓔ design-skill cascade 시 목록과 함께 전달한다. design-skill의 C9(시각 전환) 규칙이 자동 발동하도록 만드는 브릿지.

### 12-1. 시각소스 10종 신호

| 코드 | 신호 | 원본 예시 | → 전환 타겟 |
|------|------|----------|------------|
| V1 | 수치 비교 (2+ 항목) | "A안 4/10, B안 5/10, C안 8/10" | 바차트·레이더 |
| V2 | 시간축 이벤트 (3+ 시점) | 2026~2032 연도별 사건 | 타임라인 |
| V3 | 순차 프로세스 (2+ 단계) | "MOU → 업무위탁 → 주주간 → 운영" | 플로우차트 |
| V4 | 순환 구조 | "교육→매칭→프로덕션→콘텐츠→교육" | 순환 SVG |
| V5 | 2축 분류 (2×2) | "영향력 vs 대체가능성 4대 권리" | 매트릭스 |
| V6 | 핵심 단일 수치 | "연 200~400억", "10년 2000~3000억" | big-number |
| V7 | 계층·트리 | 상위→하위 조직도·분류 | 트리 다이어그램 |
| V8 | 관계망 | 엔티티 간 연결 | 노드 그래프 |
| V9 | 비율 분배 | 100% 내 구획 | 도넛·스택바 |
| V10 | 퍼널 | 진입→전환 감소 | 퍼널 SVG |

### 12-2. 감지 프로토콜

1. ⓑ 블록 설계 중 각 블록을 위 10신호로 스캔
2. 감지되면 블록 메타에 `[V#]` 태그 부착 (내부 표기, 본문 노출 ✗ — STEALTH)
3. ⓔ cascade 시 design-skill에 "시각소스 목록 {블록ID: V#}" 전달
4. design-skill은 visualization-html.md 로드 + 해당 블록을 시각요소로 전환

### 12-3. 전달 포맷 (내부 메타)

```
블록 B3 [V1 바차트]: 히트점수 4방향 비교
블록 B5 [V2 타임라인]: 10년 로드맵
블록 B7 [V3 플로우]: 4단계 계약
블록 B9 [V6 빅넘버]: 연 매출 규모
```

### 12-4. 시각소스 없음 판정

- 순수 서사, 단일 인용, 메모형 문서 → 시각소스 0개 → 전달 없음, C9 면제
- 문서 전체가 표 1개 → V1 단일 전달 (도넛·바차트로 전환 권고)

### 12-5. 과잉 방지

- 시각소스 n개 감지 시 전환 상한 = n. 초과 시 서사·맥락 블록 복원
- 문서 500행+ 에서도 시각요소 8개 이내 권장 (인지 부하 방지)

## 외부 함정 (Gotchas)

| 함정 | 대응 |
|------|------|
| 타 스킬 로직을 변경 | cascade는 출력 형태만 변환. 분석 자체는 타 스킬 관할 |
| spine 없이 블록설계만 적용 | spine 추출이 선행. 실패 시 참조형 폴백 |
| 부록 제거 | 부록은 삭제 ✗. 본문에서 빼되 문서 내 보존 |
| 3티어 게이트 무시하고 일률 풀 적용 | 허브 §0 용도×길이 매트릭스 확인 필수. 내부 20행↓ = 패스스루 |
| ⓑⓒ를 2패스 재작성 | 통합 단계에서 1회 작성으로 동시 적용. 2패스 ✗ |
| spine 강제로 데이터 문서 왜곡 | 순수 데이터/리스트/다중 동등 결론 → 참조형 폴백 |
| QC를 일률 4축 전수 | 티어별 QC 범위 참조. 경량=①숫자+③VOCAB만, 패스스루=스킵 |
