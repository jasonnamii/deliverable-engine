# BAN_LEXICON — AI 흔적 사전 (v3.0 신규, INV 16)

산출물에서 grep 적발 시 차단·평문변환·재출력. 형 코퍼스 23개 PDF·1MB 실측으로 0회 검증된 어휘 위주.

본 사전은 SKILL.md INV 16의 단일 권위. `scripts/humanize_check.py`는 이 사전의 *보조*.

---

## 1. 본질 5줄

| | |
|---|---|
| **RULE** | AI가 흘리는 5대 흔적 단어 산출물 노출 = FAIL. 형 코퍼스 1MB 실측으로 0회 검증된 어휘 우선 차단 |
| **판정** | "이 단어, 형 코퍼스 1MB 안에서 한 번이라도 등장하나?" — NO → 차단 |
| **ALLOW** | 업계 전문용어 · 고유명사 · 법조문 · 직접 인용 |
| **CONVERT** | 적발 → 평문 풀어쓰기. 동의어가 아닌 *형 시그니처 어휘*로 재구성 |
| **SELF_CHECK** | 산출 직전 자체 grep. 1개라도 적발 = 차단·재작성 |

## 9. 자체검사 절차

```bash
# 1차: 사전 grep
python scripts/humanize_check.py {산출파일경로}

# 2차: LLM 자체 판정
"이 문장, 형 코퍼스 1MB 안에 한 번이라도 등장할 만한가?"
NO → 차단·재작성
```

**부분치환 금지.** 적발 단어만 바꾸지 말 것. 사고 자체가 AI식이어서 부분치환만으론 자연어가 안 됨. **전수 재작성**으로 사고 자체를 바꾼다.

---

## 10. 우선순위 — BAN_LEXICON vs 다른 규칙

BAN_LEXICON은 **최상위 게이트**. 다른 규칙(예: CEW 한 줄, 압정 비율)이 BAN 단어를 요구하면 그 규칙을 자연어로 풀어쓴다. BAN 단어가 통과하는 일은 없다.
