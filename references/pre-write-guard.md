# PRE_WRITE_GUARD — 사전 작성 가드레일 (v3.1 신규, INV 19)

페이퍼엔진 작동 메커니즘 전환의 핵심. 사후교정형(LLM이 일단 쓴 뒤 9패스 압축) → **사전생성형(첫 단어부터 페이퍼엔진 룰 강제)**.

본 문서는 SKILL.md INV 19의 단일 권위. §B 작성 단계 진입 전 강제 적용.

---

## 1. 본질 5줄

| | |
|---|---|
| **RULE** | §B 작성 진입 전 5종 룰 강제 적용. 첫 단어 쓰기 *전*에 룰 박힌 상태로 작성 시작 |
| **메커니즘** | 사후교정 ✗ (1번) · 사전생성 ○ (2번). LLM 자유생성 → 9패스 압축 = 1번. 룰 박고 작성 → 안전망 검사 = 2번 |
| **5종** | ① 단문 8~20자 ② 시그니처 자연주입 ③ BAN 사전회피 ④ 문장당 1명제 ⑤ 문단 ≤80자 |
| **자가검사** | 작성 *직전* LLM 자기검열: "이 문장, 형 코퍼스 1MB에 등장할 만한가?" NO → 재생성 |
| **위반 시** | §C 9패스가 *마지막 안전망*. PRE_WRITE_GUARD 충실 = §C 발동 최소화 |

## 2. 5종 룰 상세

### 룰 1 — 단문 8~20자 (첫 단어부터)

**적용:** 모든 문장. 첫 단어 쓸 때부터 문장 길이 의식.

| ❌ AI식 (25자+ 종속절 다중) | ✅ 형식 (8~20자) |
|---|---|
| "이 시장은 다양한 측면에서 지속적으로 성장하고 있다고 할 수 있다." | "시장이 자란다. 지난 3년 +45%." |
| "고객은 가격보다는 시간적 여유를 중시하는 경향이 점차 늘어나고 있다." | "고객은 시간을 산다. 가격이 아니다." |
| "조직 차원에서의 체계적인 접근이 필요한 시점이라고 판단됩니다." | "조직이 움직여야 한다. 지금이다." |

**검사:** 25자 초과 = 즉시 분절 결정. 한 문장에 **,** 2개 이상 = 분리 후보.

### 룰 2 — 시그니처 자연주입 (문단당 1+ 권장, 강제 ✗)

**적용:** 문단 단위. 형 코퍼스 1MB·19,874줄에서 196회 등장한 어휘를 *자연스럽게* 흘리기.

| 어휘 | 빈도 | 쓰임 예 |
|---|---:|---|
| 직접 | 67 | "고객이 직접 결정한다" |
| 그냥 | 28 | "그냥 두면 떨어진다" |
| 왜 | 18 | "왜 지금인가" |
| 어떻게 | 18 | "어떻게 작동하나" |
| 정말 | 13 | "정말 필요한 건 X다" |
| 확실히 | 10 | "확실히 다른 결" |
| 진짜 | 10 | "진짜 차별점은 Y" |
| 사실 | 9 | "사실 핵심은 Z" |
| 딱 | 9 | "딱 한 가지" |
| 근데 | 9 | "근데 여기서 막힌다" |
| 솔직히 | 5 | "솔직히 안 된다" |

**금기:** 무리하게 끼워 넣지 말 것. 자연스럽지 않으면 **빼고 다른 시그니처로**. 강제 ✗.

### 룰 3 — BAN 사전회피 (토큰 선택 시점)

**적용:** 단어 선택 매 순간. references/banwords-lexicon.md 60+ 어휘를 *처음부터 안 고르기*.

| BAN 후보 | 사전 대체 |
|---|---|
| "결론적으로" | (삭제 — 결론은 첫 줄에) |
| "본질적으로" | "결국·진짜로" |
| "구체적으로" | (실명 직시) |
| "한편" | (삭제 또는 "그런데") |
| "라고 할 수 있다" | "~이다" |
| "할 수 있다" | "~한다·~된다" |
| "~에 대한" | (격조사 축약) |
| "~을 통해" | "~로·~에서" |
| "다양한" | (구체 숫자·실명) |
| "지속적으로" | "계속·꾸준히" |

**메커니즘:** 9패스 사후 grep ✗ → 토큰 생성 시점 사전 회피. 처음부터 안 고르면 9패스 발동 없음.

### 룰 4 — 문장당 1명제

**적용:** 문장 단위. 한 문장 = 한 생각. 복문·접속절 회피.

| ❌ 복문 (다중 명제) | ✅ 단문 분절 |
|---|---|
| "시장이 성장하고 있고 경쟁이 심화되고 있어 차별화가 필요한데, 우리는 그 답을 찾았다." | "시장이 자란다. 경쟁이 거세다. 차별화가 필요하다. 답은 X다." |
| "고객은 가격보다 시간을 중시하기 때문에 우리는 빠른 배송을 핵심 전략으로 삼아야 하며, 이는 매출 증대로 이어질 것이다." | "고객은 시간을 산다. 빠른 배송이 답이다. 매출이 따라온다." |

**검사:** 접속어 ("그리고·하지만·그래서·~인데·~기 때문에") 1문장에 2개+ = 분리.

### 룰 5 — 문단 ≤80자

**적용:** 문단 단위. 한 문단 80자 초과 = 분절 후보.

**근거:** 형 코퍼스 PT 슬라이드 본문 평균 60~80자. 한국어 단문 미학.

| ❌ 80자+ 단락 | ✅ ≤80자 분절 |
|---|---|
| "이번 캠페인은 다양한 채널을 통해 진행되며 SNS·유튜브·포털·OOH 등 4대 매체에 동시 집행되어 통합적인 인지도 상승을 노리는 한편 동시에 매출 전환까지 잡는 이중 트랙 전략입니다." | "4대 매체 동시 집행. SNS·유튜브·포털·OOH. 인지도와 매출 둘 다 잡는다." |

**검사:** `wc -m` 으로 단락별 글자 수. 80 초과 = 분절.

## 3. 토큰별 자체검증 (룰 6 — 메타룰)

작성 *직전* LLM이 자기 검열:

> **"이 문장, 형 코퍼스 1MB(23개 PDF·19,874줄)에 한 번이라도 등장할 만한가?"**

- **YES** → 작성 진행
- **NO** → 재생성. 어디가 안 어울리는지 5종 룰 다시 적용

**판정 보조:**
- "이 단어, 형이 자주 쓰나?" (시그니처 카운트)
- "이 단어, 형 코퍼스 0회 단어 아닌가?" (BAN 카운트)
- "이 문장 길이, 형 평균(8~20자) 안인가?"

## 4. §B 작성 단계 적용 흐름

```
③ §B 작성 진입
→ §B-PRE: 5종 룰 + 룰 6 메타 자기검열 활성화
→ §B-1 Headline 생성  ← 룰 1·3·4 강제
→ §B-2 의사결정 1줄    ← 룰 1·3 강제
→ §B-3 Lead 생성        ← 룰 1·2·3·4 강제 (5W1H 단문 분리)
→ §B-4 Body 생성        ← 룰 1·2·3·4·5 전수 강제 (HERO + 단락)
→ §B-5 Pin↔Body 매핑    ← 룰 4 강제 (1압정 1명제)
④ §C 9패스 ANTI_BLOAT — 마지막 안전망 (사전 잘 잡혔으면 0패스)
```

PRE_WRITE_GUARD 충실 시 §C 9패스 평균 적용 패스 수 ≤ 2 (현재 v3.0 평균 ~7).

## 5. §B 단계별 생성 룰 (구체)

### §B-1 Headline 생성 룰

**형식:** 결론 주장형 1줄. 형 코퍼스 패턴.

| ✅ 형식 (주장형) | ❌ AI식 (설명형) |
|---|---|
| "양평DNA는 셋이다." | "양평DNA에 대한 종합 분석" |
| "고객은 시간을 산다." | "고객 행동 패턴의 의미와 시사점" |
| "1기는 7월이 아니라 9월이다." | "1기 일정 검토 결과" |
| "X는 Y가 아니라 Z다." | "X와 Y의 비교 검토" |

**금지 패턴:**
- "~에 대한·~의 의미·~의 시사점·~분석·~검토" 종결 → 즉시 재작성
- 의문문 종결 → 단정형으로 전환
- 25자 초과 → 분절

### §B-3 Lead 생성 룰

**형식:** 5W1H 또는 결론+근거+사례. ≤5문장 강제.

| 권장 패턴 | 예 |
|---|---|
| 결론 + 근거 1줄 | "양평DNA는 셋이다. 정원·물·문학." |
| 수치 + 출처 박스 | "정원인구 13.5만 (KOSIS 2024). 1년 +12%." |
| 인용박스 (FGI 직인용) | "고객 증언: '여기 다시 오는 이유는 풍경이다.'" |

**룰:**
- 1문장당 1수치·1고유명사 의무
- "~할 수 있다고 본다·것으로 판단된다" 금지
- 5문장 초과 = Body로 이관

### §B-4 Body 생성 룰

**형식:** 소제목 + HERO 1줄 + 본문. 본문은 단락 ≤80자.

```
## [소제목 — 명사형 또는 X는 Y다 형]

[HERO 1줄 — 압축 한 문장. 담담한 사실 진술]

[본문 단락 1 — ≤80자, 단문 분절, 시그니처 자연 주입]
[본문 단락 2 — ≤80자, 수치+출처 박스 또는 인용박스]
[본문 단락 3 — ≤80자, 결론 한 번 더 받침]
```

**룰:**
- HERO 1줄 = 룰 1·3·4 강제. 마침표 종결. 두 문장 = FAIL
- 단락 ≤80자 = 룰 5 강제. 초과 시 분절
- 명사형 맺음 권장 (형 코퍼스 패턴: "이유는 X·핵심은 Y·차이는 Z")
- 반복 리프레인 권장 (형 코퍼스 패턴: "싸고 쉽게 / 싸고 쉽게")

### §B-5 Pin↔Body 매핑 룰

각 압정 주장 = Body 단락 1개와 1:1 매핑. 한 문장 한 명제 (룰 4) 강제.

## 6. 자체검사 절차 (산출 직전)

1. Headline 1줄 — 주장형인가? 25자 이하인가? BAN 어휘 있나?
2. Lead — 5문장 이하? 1문장당 1수치·1고유명사? "~할 수 있다고 본다" 있나?
3. Body 각 섹션 — HERO 1줄 있나? 단락 ≤80자? 시그니처 1+?
4. 전체 — 형 코퍼스 1MB에 등장할 만한가? (LLM 자기검열)

위반 1+ → 해당 부분 재작성. 부분치환 ✗.

---

## 7. 우선순위 — PRE_WRITE_GUARD vs 다른 규칙

PRE_WRITE_GUARD는 §B 작성 단계의 **사전 강제 게이트**. 다른 규칙(예: HERO 형식·CEW)이 PRE_WRITE_GUARD와 충돌하면 PRE_WRITE_GUARD 우선.

예: HERO 1줄이 "결론적으로 X이다" 같이 BAN 어휘 포함 → PRE_WRITE_GUARD 룰 3 위반 → 재작성. HERO 형식만 만족하고 BAN 통과 = 무효.

## 8. Gotchas

| 함정 | 대응 |
|---|---|
| 사후 9패스에 의존하는 관성 | PRE_WRITE_GUARD = §B 진입 전 *룰 박힌 상태*로 시작. §C는 마지막 안전망 |
| 시그니처 무리하게 끼우기 | 자연스럽지 않으면 빼라. 강제 ✗ |
| 단문 강박으로 의미 누락 | 단문은 *호흡*. 사실·수치는 절대 보존 |
| Headline에 HERO 룰만 적용 (BAN 통과) | PRE_WRITE_GUARD 룰 3 우선. BAN 어휘 = 즉시 재작성 |
| LLM 자기검열 건너뛰기 | "형 코퍼스 1MB에 등장할 만한가?" 매 문장 1회 |
| 80자 단락 룰을 모든 텍스트에 적용 | 데이터 박스·인용박스·수치 표 면제. 산문 단락에만 |
