# File Operations — 네이밍·버전·저장·시각문법

산출물 파일의 운영 규칙. 네이밍 → 버전 → 덮어쓰기 방지 → 저장 경로 → 시각문법.

---

## 네이밍 · 버전

```
네이밍 ::= 맥락_주제_v{M}.{m}.확장자
  ✗ 카테고리 단독("보고서.md")
버전 ::= 형 컨펌 전=버전미부여 | 컨펌 후=v0.1~
  .9 → .10
```

## 신규 작성 사전 체크 게이트

**Write 실행 전 기존 파일 존재 확인 필수.** "새 문서 작성"이라도 대상 경로에 동일·유사 파일이 이미 있으면 신규가 아니라 수정이다.

```
Write 호출 직전 → 대상 경로에서 ls/Glob 확인:
  ①정확히 같은 파일명 존재? → 신규 아님 → 아래 "기존 문서 수정 판정"으로 분기
  ②같은 주제의 유사 파일명 존재? → 형에게 확인 ("기존 파일 X가 있습니다. 새로 만들까요, 수정할까요?")
  ③해당 없음 → 신규 작성 진행
미확인 Write = FAIL
```

## 기존 문서 수정 판정

| 판정 | 조건 | 행동 |
|------|------|------|
| 부분수정 | 수정 범위 ≤ 전체의 30% + 섹션 구조 유지 | 기존 파일 Edit + 파일명 버전 Minor 범프 |
| 실질적 재작성 | 수정 범위 > 30% 또는 섹션 구조 변경 또는 핵심메시지 변경 | **새 파일 생성 필수** + 이전 파일→_archive/ |

**덮어쓰기 절대금지:** 기존 문서에 Write(전체 덮어쓰기)를 실행하는 것은 부분수정·재작성 무관하게 FAIL.

## 저장 경로 (SAVE_CHAIN)

```
①프로젝트폴더 → ②컨텍스트폴더 → ③VAULT/_inbox(DC우선) → ④mnt/outputs(최종fallback+"이동필요"안내)
```

- 기본 포맷: .md (형 포맷명시시 해당포맷)
- 저장 후 경로 1줄 보고 필수
- 세션브리핑은 이 경로 적용 제외 (session-briefing 스킬 관할)

## 시각문법

산출물의 시각 디자인은 **design-skill이 단일 권위**. design-skill이 포맷별로 적절한 하위 규칙을 cascade한다.

**CASCADE 위치:** §0 ⓕ가 유일한 design-skill 호출 지점. 여기서 별도 재호출 불필요 — §0 ⓕ 미실행 시만 FAIL.

| 포맷 | 디자인 권위 | 비고 |
|------|------------|------|
| .md | design-skill → protocol-pretty.md (이쁘니) | 콜아웃·강약·밀도교대·QC |
| .html | design-skill → format-html.md | 4층 엔진 + CORE |
| .md 내 div | design-skill + html-div-style | html-div-style은 div 래핑 안전성만 |
| 기타 (.pptx/.docx/.pdf) | design-skill → 해당 format spoke | |

시각화시 8유형에서 선택: 플로우·구조·매트릭스·타임라인·퍼널·상태전이·비교·인과
- Mermaid → 형 명시요청시만. 자율발동 ✗
- ✗ 코드블록 ASCII를 산출물에 사용

산출물 엔진 ③QC 시 디자인 검사는 design-skill의 해당 포맷 QC를 적용한다 (md → 이쁘니 QC ①~⑧).
