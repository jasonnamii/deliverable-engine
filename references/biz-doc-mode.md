# BIZ_DOC — 비즈니스 문서 모드 (v4.4 신규, INV 30)

§A-2 DOC_TYPE 5번째. 외부 독자(투자자·관계자·파트너) 대상 비즈니스 문서 본문 전용. DELIVER(보도·1pager)·UTTER(카피·매니페스토)·CONVERGE(다소스 종합) 어디에도 안 맞는 영역.

---

## 1. 본질 1줄

**RULE:** 외부 독자 비즈니스 문서 본문 = 완결문·연결조사·풀어쓰기. 압축 강제 OFF. 카피·매니페스토 블록만 UTTER로 명시 면제.

---

## 2. 정의 — 언제 BIZ_DOC인가

| ✅ BIZ_DOC | ✗ 다른 모드 |
|---|---|
| 투자 제안서·IR 본문 | 보도·1pager → DELIVER |
| 전략 설명서·기획안 본문 | 매니페스토·헤로카피 → UTTER |
| 사업 보고서·플레이북 본문 | 진단서 → DIAGNOSE |
| 외부 제출 문서 본문 | 비교·우열 → EVALUATE |
| 파트너 제안 본문 | 다소스 종합 → CONVERGE |

**핵심 분기점:** *외부 독자가 한 번 읽고 이해해야 하는가?* YES = BIZ_DOC. 내부 메모·결정 로그·진단서는 압축형 OK.

## 3. §B-PRE 분기 (BIZ_DOC 모드)

| 룰 | DELIVER 본문 | **BIZ_DOC 본문** | UTTER 블록 |
|---|---|---|---|
| ① 단문 8~20자 | 강제 | **20~40자 권장. 25자+ 분절 강제 OFF** | 면제 |
| ② 시그니처 자연주입 | 권장 | **OFF** (외부 독자에게 형 시그니처 ✗) | 강제 |
| ③ BAN 회피 | 강제 | 강제 | 면제 일부 |
| ④ 1문장 1명제 | 강제 | **연결조사·종속절 ALLOW** (단, 한 문장 명제 2개+ = FAIL) | 면제 |
| ⑤ 문단 ≤80자 | 강제 | **문장 ≤60자·문단 ≤200자** | 면제 |
| ⑥ 격식 LV | LV2~3 | **LV3 강제** (외부 독자) | LV1 |
| ⑦ 작업라벨 회피 | 강제 | 강제 | 강제 |
| ⑧ KIWI 문법 | 강제 | 강제 | 강제 |
| ⑨ Intent-Quote | 권장 | **권장** | 강제 |
| ⑩ ABSTRACT-BAN | 강제 | **강제** (UTTER 블록만 ALLOW — block-level 분기 그대로) | 면제 |

**핵심 차이:**
- ① 단문 강제 해제 → **완결문·연결조사 ALLOW**
- ② 형 시그니처(직접·그냥·왜·정말·진짜·근데·솔직히) **OFF** — 외부 독자 위화감
- ④ 종속절·연결어 ALLOW — 단, 1문장 = 1명제 원칙 유지
- ⑤ 문단 200자까지 OK — 외부 독자에겐 호흡 긴 문장이 자연

## 6. ❌WRONG / ✅CORRECT — c8 case 박제

**케이스 1: WHY 시장 진단**
```
❌ BIZ_DOC 본문에 카피 톤:
   "콘텐츠 공급 과잉 + 알고리즘 단축화. 창작자 자아 소진이 신규 시장 신호. 협업·도시 자리 부재가 구조적 공백."
   → 명사구 나열·조사 누락·외부 독자에게 메모처럼 읽힘

✅ BIZ_DOC 본문 (완결문·연결조사·풀어쓰기):
   "콘텐츠 공급 과잉과 알고리즘 피로는 새로운 플랫폼 전환 신호입니다.
    창작자 자아 소진은 단순 현상이 아니라 새로운 시장 기회의 시작입니다.
    현재 시장에는 협업 구조와 오프라인 기반 창작 장면이 부재합니다."
```

**케이스 2: 캠페인 실행 설명**
```
❌ BIZ_DOC 본문에 카피 톤:
   "광고비가 아닙니다. 분기점입니다."
   → 카피 톤. 외부 독자에겐 의미 단절

✅ BIZ_DOC 본문:
   "이 비용은 단순 광고비가 아닙니다.
    초기 네트워크 효과를 만드는 투자입니다."

✅ UTTER 카피 블록 (매니페스토 안):
   "광고비가 아닙니다. 분기점입니다." ← UTTER 블록 = ALLOW
```

**케이스 3: KPI 설명**
```
❌ BIZ_DOC 본문:
   "가입 후 7일 안에 1회 작업 행동."
   → 메모. 외부 독자에게 서술 불완전

✅ BIZ_DOC 본문:
   "핵심 KPI는 가입 이후 7일 내 첫 창작 행동 발생률입니다."
```

## 7. 같은 문서 안 BIZ_DOC + UTTER 혼합

비즈니스 문서엔 본문(BIZ_DOC) + 매니페스토 블록(UTTER) + 카피 풀(UTTER)이 흔히 공존. **block-level 분기로 처리.**

```
[기획안 본문 = BIZ_DOC]
"이 캠페인의 핵심 가설은 1만 활성 사용자 도달입니다.
 도달 시점부터 자생적 네트워크 효과가 발생하는지 검증합니다."

[매니페스토 블록 = UTTER 명시]
<div class="manifesto-block">
"광고를 만들지 않았습니다. 사건을 만들었습니다."
</div>

[카피 풀 = UTTER 명시]
<div class="copy-block">
"인스타에선, 묻혔습니다. 여기선, 보입니다."
</div>
```

자동 인식 마커는 `abstract-ban.md §4` 그대로.

## 10. v4.4 핵심 메시지

기존 5종 (DELIVER·DIAGNOSE·EVALUATE·CONVERGE·UTTER)으론 **외부 독자 비즈니스 문서 본문**의 영토 부재. 카피처럼 압축되거나(DELIVER 침범) 매니페스토처럼 박웅현 톤(UTTER 침범)으로 변질. BIZ_DOC = 그 영토의 단일 정본.
