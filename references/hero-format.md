# HERO 형식 — 기본 모드 (v2.6 신규, INV 14)

산출물의 모든 섹션 첫 줄을 압축 한 문장으로 시작한다. 첫 줄만 차례대로 읽어도 문서 전체의 방향성을 파악할 수 있다. DOC_TYPE(전달·진단·평가·수렴)·RENDER(MD·HTML)·MODE(S·M·L) 위에 얹히는 형식 층.

---

## 1. 본질 4줄

| | |
|---|---|
| **RULE** | 모든 본문 섹션의 첫 줄 = 압축 한 문장 (HERO). HERO만 차례로 읽으면 분석 → 핵심 → 아이템들의 서사가 흐름 |
| **톤** | 담담. 선언적·멋짐 금지. 사실 진술 톤 |
| **분리** | HERO = 형식 / DOC_TYPE = 내용. NYT·모래시계·역피라미드 등 내용 구조와 직교. 어떤 종류든 HERO를 입을 수 있음 |
| **자가검사** | 산출 직전 HERO 줄만 추출해 차례로 읽기. 서사 흐름 깨짐 → 재작성 |

---

## 2. HERO 한 줄 룰

| 항목 | 룰 |
|---|---|
| 위치 | 각 본문 섹션의 첫 줄 (제목 바로 다음) |
| 길이 | 한 문장. 평균 25~50자. 두 문장 이상 = FAIL |
| 어조 | 평서문·사실 진술 |
| 금지 | 선언·구호·외침·반복강조·말줄임 |
| 허용 | 단언·관찰·사실·필요·결정 |
| 마침 | 마침표로 끝남 |

---

## 3. 서사 흐름 — 분석 → 핵심 → 아이템

HERO 줄을 묶음 단위로 본다. 각 묶음(보통 한 章·Part)에는 다음 구조가 들어간다.

| 단계 | 역할 | 위치 |
|---|---|---|
| **분석** | 왜 이 묶음을 다루나·무엇이 흔들렸나 | 묶음 첫 HERO |
| **핵심** | 이 묶음의 도달점·원리·결정 | 분석 다음 HERO |
| **아이템** | 도달점을 구성하는 요소들 | 핵심 이후 HERO들 |

### 묶음이 셋인 문서의 HERO 흐름 예

```
[묶음 1]
H1. (분석) 한 상품으로 모든 학생을 담기 어렵다는 것이 분명해졌다.
H2. (핵심) 분리의 기준은 연령이 아니라 학습 목적이다.
H3. (아이템) 세 트랙은 같은 캠프의 옵션이 아니라 서로 다른 상품이다.
H4. (아이템) 트랙 안에는 영어 수준별 분반이 한 단계 더 들어간다.
H5. (아이템) 같은 공간을 함께 쓰되 동선은 분리한다.

[묶음 2]
H6. (분석) 7월에 바로 1기 60명 모집을 시작하기는 어렵다.
H7. (핵심) 5월 시범, 6월 모집, 7월 본 캠프, 8월 이후 지속 운영의 다섯 단계로 나눈다.
H8. (아이템) 시범은 영상 촬영 이벤트가 아니라 일곱 트랙이 동시에 움직이는 작업이다.

[묶음 3 — 합성]
HX. (핵심) 트랙 분리는 지속 운영으로 자연스럽게 이어진다.
HY. (아이템) 강사 24명은 두 단계에서 다른 역할로 환원된다.
```

마지막 묶음(합성·결론)은 분석 생략 가능. 묶음 첫 HERO가 핵심 또는 결론 위치를 잡으면 통과.

---

## 4. 본문 (HERO 아래)

본문은 DOC_TYPE에 따른다. 전달형은 NYT 압정 본문, 진단형은 5Whys + 처방, 평가형은 등급 + 근거, 수렴형은 다소스 매트릭스 + 논증. HERO는 형식, 본문은 내용.

본문 분량은 섹션마다 다를 수 있다. 짧은 섹션은 HERO 한 줄 + 표 1개, 긴 섹션은 HERO 한 줄 + 배경 + 표·근거 + 실행 정보.

---

## 5. 담담 톤 — 선언적·멋짐과의 차이

| 선언적·멋짐 (금지) | 담담 (통과) |
|---|---|
| 한 상품으로 팔던 시대는 끝났다. | 한 상품으로 묶어 운영하기 어려워졌다. |
| 결과는 하나, 트랙은 셋. | 트랙은 셋이고 각각 다르게 운영한다. |
| 준비된 구조만 살아남는다. | 준비가 먼저고 행동은 그 다음이다. |
| 7월에 모든 것이 결정된다. | 7월의 결과는 7월에 만들어지지 않는다. |
| 강사는 가르치는 사람이 아니다. | 강사의 역할은 가르치는 것이 아니라 방향을 잡는 것이다. |

판정: HERO를 소리내어 읽었을 때 외침 또는 카피 느낌이 들면 담담 실패. 관찰 또는 사실 진술 느낌이면 담담 통과.

## 6. 산출물 어조 룰 (INV 15 연동)

HERO만이 아니라 본문 전반에 적용되는 어휘 룰.

### 금지 어휘

| 분류 | 금지 | 대체 |
|---|---|---|
| 인물 호칭 | 피디님·작곡가·박사·형·내가·나·우리 | 역할명 (캠프 대표·학술 디렉터·운영 담당) 또는 발화 주체 명시 생략 |
| 버전 라벨 | v3.2·v1.1·SoT MD·{문서명}_v{N} | "기존 프로그램·기존 사업기획" |
| 작업 약자 | LTV·NPS·MoM·CAC·ARPU | 풀어쓰기 (지속 수익 운영·재등록 의향·전월 대비 등). 단 BEP·KPI·MECE·MVP는 업계 표준이라 한 번 정의 후 사용 가능 |
| 장사 어휘 | 팔다·판다·세일즈·구매·결제까지 데려가다·전환·만석 | 운영·진행·합류·등록·결정 |
| 작업 라벨 | 앞단·뒷단·도달점·주변점·서포트·단위·핑퐁·화학·X축 | 평문 풀어쓰기 (앞부분·뒷부분·결론·근거 등) |

### 허용

- 트랙명·플랫폼명·작품명 같은 고유명사
- 업계 표준 정착 약자(BEP·KPI·MECE·MVP) — 한 번 정의 후 사용
- 법조문 번호

### 자가검사 절차

1. HERO 줄만 추출 → 위 금지 카탈로그와 대조. 1개라도 발견 → 재작성.
2. 본문 전체 → 같은 대조. 발견 시 평문 풀어쓰기로 전수 치환. 부분 치환 금지.

## 7. HERO + DOC_TYPE 결합 예

### 전달형 + HERO

```
H1. (분석) 매출이 분기마다 떨어지고 있다.
   본문: KOSIS 2024 4분기 -12%, 전년 동기 대비. 핵심 채널 셋 모두 하락.
H2. (핵심) 떨어진 자리는 가격이 아니라 신규 유입이다.
   본문: 고객 행동 데이터 ...
H3. (아이템) 신규 유입 채널 셋 중 둘이 멈춰 있다.
   본문: ...
```

### 진단형 + HERO

```
H1. (분석) 1기 등록 이탈률이 동종 캠프보다 높다.
   본문: 후기 자산 부재·문의 응대 지연·결제 페이지 오류 셋이 동시에 작용.
H2. (핵심) 원인의 60%는 결제 페이지 단일 지점에서 발생한다.
   본문: 5Whys ...
H3. (아이템) 결제 페이지를 단일 안정 버전으로 교체한다.
   본문: 처방 + 리스크 ...
```

### 수렴형 + HERO

```
H1. (분석) 다섯 편의 리서치가 같은 결론을 다른 각도에서 가리킨다.
H2. (핵심) 진짜 차별점은 결과물의 종류가 아니라 결과물의 소유 구조다.
H3. (아이템) 결과물은 학생 이름으로 발행되어야 한다.
H4. (아이템) ...
```

DOC_TYPE은 본문 구조를, HERO는 첫 줄을 결정한다. 둘은 직교한다.

## 8. Gotchas

| 함정 | 대응 |
|---|---|
| HERO를 카피처럼 멋지게 작성 | 담담 룰. 선언·외침 금지 |
| HERO 두 문장 | 한 문장 강제 |
| HERO 빠진 섹션 | INV 14 FAIL |
| 분석 없이 아이템 나열 | 묶음 첫 HERO에 분석·핵심 의무 (마지막 묶음 분석은 면제) |
| 본문이 HERO와 무관 | 본문 첫 단락은 HERO를 받쳐야 함 |
| HERO 줄을 표·인용박스로 시작 | HERO는 평문 한 문장. 표는 그 다음 |
| 인물 호칭 누출 (피디님·박사) | INV 15 → 역할명 |
| 작업 약자 (LTV 남발) | INV 15 → 풀어쓰기, 업계 표준만 통과 |
| 장사 어휘 (팔다·전환·만석) | INV 15 → 운영·합류·등록 |
