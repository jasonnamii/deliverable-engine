# DOC_TYPE_ROUTER — 문서 종류별 분기 (v2.3 신규)

길이 분기(MODE_S/M/L)는 *분량*만 본다. 문서의 *목적·논리 부담*은 다르다. v2.3부터 종류별로 압정 비율·필수 의무를 차등.

---

## 5종류 정의 (v4.4 BIZ_DOC 추가)

| 종류 | 정의 | 압정 비율 | 핵심 의무 |
|------|------|-----------|----------|
| **전달형 (DELIVER)** | 보도·요약·1pager — 결론을 빠르게 전한다 | 1:7:2 | NYT 압정·삭제·밀도 |
| **진단형 (DIAGNOSE)** | 원인 → 처방 — 무엇이 잘못됐고 어떻게 고치는가 | 1:6:3 | 진단 본문 + 처방 명시 |
| **평가형 (EVALUATE)** | 비교·등급·우열 — 옵션 간 우열 판정 | 1:5:4 | **모든 등급 옆 근거 1줄 의무** |
| **수렴형 (CONVERGE)** | 다소스 통합·논증 — N편 리서치를 1결론으로 | 1:4:5 | **다소스 교차비교 + 논증 본문 의무** |
| **🆕 비즈니스형 (BIZ_DOC)** | 외부 독자 비즈문서 — 투자제안·전략설명서·기획안 본문·IR 본문 | 1:5:4 | **완결문·연결조사·풀어쓰기·LV3 강제. 정본 = `biz-doc-mode.md`** |

---

## 종류 판정 (트리거 키워드)

| 사용자 의도 | 종류 |
|------------|------|
| "보고서·요약·1pager·뉴스" | DELIVER |
| "진단해줘·원인 분석·처방·뭐가 문제·어떻게 고쳐" | DIAGNOSE |
| "비교해줘·우열·등급·매트릭스·옵션 평가" | EVALUATE |
| "통합·종합·수렴·N편 정리·리서치 종합" | CONVERGE |
| **🆕 "투자제안·전략설명서·기획안 본문·IR 본문·외부제출문서·비즈문서·완결문으로·풀어써·BIZ_DOC"** | **BIZ_DOC** |

복합 시 **수렴형 > 평가형 > 진단형 > BIZ_DOC > 전달형** 우선 (논리 부담 큰 쪽).
**예외:** "투자자·외부 독자·파트너용" 명시 → BIZ_DOC 우선.

## 종류별 골격

### DELIVER (전달형)
```
[Headline] 결론 1줄
[Lead] 5W1H 요약 (5문장 이하)
[Body] 섹션 3~5개·중요도 내림차순
[Kicker] 한 줄 종결
```
- 압정 1:7:2 (현 NYT 표준)
- 본문 = 사실 나열 + 짧은 해석

### DIAGNOSE (진단형)
```
[Headline] 진단 결론 1줄 (예: "X가 매출 40% 깎는다")
[Lead] 증상 + 원인 + 처방 요약
[Body §1 증상] 구체 사실·수치
[Body §2 원인] 5Whys·인과 사슬
[Body §3 처방] 액션 1~3개·실행 조건
[Body §4 리스크] 처방 부작용·대안
[Kicker] 다음 액션 1줄
```
- 압정 1:6:3 (Body의 본문 비중 ↑)
- §원인·처방 본문은 *논증* 단위 (C+E+W) 의무

### EVALUATE (평가형)
```
[Headline] 1위 + 핵심 차별점 1줄
[Lead] 비교 대상 N개·기준 M개 요약
[Body §1 비교 매트릭스] 등급표 + **등급 옆 근거 1줄 컬럼 필수**
[Body §2 우위 분석] 1위가 왜 1위인가 — 본문 논증
[Body §3 트레이드오프] 1위가 약한 면·2위가 강한 면
[Body §4 결정 조건] "X 조건이면 1위, Y 조건이면 2위"
[Kicker] 권장
```
- 압정 1:5:4 (본문 비중 더 ↑)
- **모든 등급 (강·중·약·★·점수) 옆에 근거 컬럼 의무**. 미명시 = FAIL
- "왜 이 등급?" 1줄 미작성 = 동어반복 = FAIL (양평 종합·수렴 핵심 결함)

### CONVERGE (수렴형)
```
[Headline] 통합 결론 1줄
[Lead] 통합 대상 N편·핵심 수렴점
[Body §1 다소스 교차비교 매트릭스] 편별 결론·합의·모순·빈자리 (필수)
[Body §2 수렴 논증] 왜 이 결론으로 수렴하는가 — 본문 논증
[Body §3 반대 가설 절단] 절단된 가설 N개·각각 절단 사유
[Body §4 잔존 불확실성] 통합 후에도 남은 미확정
[Kicker] 통합 결론·신뢰도]
```
- 압정 1:4:5 (본문 비중 최대)
- **다소스 교차비교 매트릭스 의무** — 단순 인덱스화 = FAIL
- 편별 *합의·모순·빈자리* 명시 의무 (3행 이상 동어반복 = FAIL)
- "수렴 논증" 본문 = C+E+W 단위 의무

## 다소스 교차비교 매트릭스 템플릿 (CONVERGE 필수)

| 편 | 핵심 결론 | 다른 편과의 합의 | 다른 편과의 모순 | 빈자리 |
|----|----------|----------------|----------------|--------|
| 편1 | | | | |
| 편2 | | | | |
| ... | | | | |

**합의 영역:** 3편+ 일치 → 통합 결론 강도 ↑
**모순 영역:** 2편 갈림 → 핵심 통찰 후보·추가 검증
**빈자리:** 어느 편도 안 다룬 영역 → §4 잔존 불확실성

| 항목 | A | B | C | A 근거 | B 근거 | C 근거 |
|------|---|---|---|--------|--------|--------|
| 기준1 | 강 | 중 | 약 | [데이터·출처 1줄] | [데이터·출처 1줄] | [데이터·출처 1줄] |
| 기준2 | ★★★ | ★★ | ★ | | | |

근거 셀이 비어있으면 = 등급 무효 = FAIL.

## 종류 ↔ 모드 매트릭스

| 길이 \ 종류 | DELIVER | DIAGNOSE | EVALUATE | CONVERGE | 🆕 BIZ_DOC |
|-------------|---------|----------|----------|----------|------------|
| ≤500자 | MODE_S | (불가·DEEP화) | (불가·DEEP화) | (불가·DEEP화) | (불가·DEEP화) |
| 500~2000자 | MODE_M | MODE_M | MODE_M | (불가·DEEP화) | MODE_M |
| 2000자+ | MODE_L | MODE_L | MODE_L | MODE_L | MODE_L |

**CONVERGE는 MODE_L 강제** (2000자 이하로 다소스 통합 ✗).
**EVALUATE·DIAGNOSE는 MODE_S 금지** (논증 본문 들어갈 자리 ✗).
**🆕 BIZ_DOC는 MODE_M·L** (MODE_S 금지 — 풀어쓸 자리 ✗·정본 = `biz-doc-mode.md`).

## v2.3 핵심 변화

- **양평 종합·수렴 같은 산출물 차단:** "압정만 적층·근거 없는 등급·다소스 인덱스화" 모두 v2.3 EVALUATE/CONVERGE에서 FAIL
- **압정 비율 차등:** 사고형 산출물(EVALUATE·CONVERGE)은 본문 비중 ↑ — 평가·논증·교차비교 들어갈 자리 확보
- **shaper-skill 단독 호출 시 종류 미명시면 DELIVER 디폴트** (기존 호환). 명시적으로 "비교해줘·통합해줘" 트리거 시 EVALUATE/CONVERGE
